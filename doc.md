لتشغيل البرنامج، اتبع الخطوات التالية بالترتيب:

### 1. ثبّت المتطلبات
افتح الطرفية (Terminal) في مجلد المشروع ونفّذ:
```bash
pip install -r requirements.txt
```
هذا سيثبّت المكتبات المطلوبة: pandas, openpyxl, pyyaml, Pillow, arabic_reshaper, python-bidi.

### 2. جهّز ملفات الإدخال
- تأكد من وجود ملف Excel في المسار `Cities/Anbar.xlsx` (أو أي ملف آخر تريد استخدامه).
- تأكد من وجود ملفات التكوين `config_layout.yaml` و `config_sizes.yaml` في المجلد الرئيسي.
- تأكد من وجود مجلد `assets` وفيه:
  - `fonts/` يحتوي على الخطوط (مثل NotoNaskhArabic-Regular.ttf و NotoNaskhArabic-Bold.ttf)
  - `mobile.png` صورة الموبايل
  - `logo.png` الشعار الرئيسي
  - `icons/` أيقونات صغيرة (اختياري)

### 3. شغّل الأمر الأساسي
من داخل مجلد المشروع، نفّذ:
```bash
python main.py --input "Cities/Anbar.xlsx" --output "./out"
```
هذا سيقرأ ملف Excel ويولّد التصاميم في مجلد `out/`.

### 4. خيارات إضافية (اختياري)
- لتوليد أول 3 سجلات فقط للتجربة:
  ```bash
  python main.py --input "Cities/Anbar.xlsx" --output "./out" --generate-sample 1 --preview
  ```
- لتوليد نسخة مع اسم المحل فقط:
  ```bash
  python main.py --input "Cities/Anbar.xlsx" --output "./out" --with-shop
  ```
- لتوليد ملفات PDF بدلاً من PNG:
  ```bash
  python main.py --input "Cities/Anbar.xlsx" --output "./out" --format pdf
  ```
- لمعاينة سريعة (صور منخفضة الدقة):
  ```bash
  python main.py --input "Cities/Anbar.xlsx" --output "./out" --preview
  ```

### 5. النتائج
ستجد التصاميم في مجلد `out/` بأسماء مثل:
```
Anbar_-_Ramadi_-_Istanbul_Plus_-_92x86_-_Mesh_-_QTY2_noshop.png
Anbar_-_Ramadi_-_Istanbul_Plus_-_92x86_-_Mesh_-_QTY2_withshop.png
```

إذا واجهت أي مشكلة، تأكد من أن جميع المسارات صحيحة وأن المكتبات مثبتة. يمكنك أيضًا تشغيل `python main.py --help` لعرض جميع الخيارات.

# دليل تكوين ملفات YAML للمشروع 📐

يستخدم المشروع ملفين أساسيين لتحديد شكل التصاميم وأحجامها:
- `config_sizes.yaml` : يحدد نسب القوالب المتوفرة (S1, S2, …) والتسامح المسموح به عند مطابقة الأبعاد.
- `config_layout.yaml` : يحدد مواقع وأحجام كل عنصر في التصميم (صورة الموبايل، الشعار، النصوص، الأيقونات، إلخ) كنسب مئوية.

كلاهما مكتوب بصيغة YAML التي يسهل قراءتها وتحريرها بواسطة أي محرر نصوص.

---

## 1. ملف `config_sizes.yaml`

هذا الملف مسؤول عن تعريف القوالب (Templates) التي سيتم مطابقة أبعاد التصميم معها. يحتوي على جزأين رئيسيين:

### 1.1 `templates`
قائمة من القوالب، كل قالب له اسم ونسبة عرض/ارتفاع (ratio). النسبة تحسب بقسمة العرض على الارتفاع (W/H).

مثال:
```yaml
templates:
  - name: S1
    ratio: 1.0      # 1:1
  - name: S2
    ratio: 2.0      # 2:1
  - name: S3
    ratio: 1.5      # 3:2
  - name: S4
    ratio: 0.5      # 1:2
  - name: S5
    ratio: 1.333    # 4:3
  - name: S6
    ratio: 0.75     # 3:4
  - name: S7
    ratio: 1.777    # 16:9
  - name: S8
    ratio: 0.5625   # 9:16
  - name: S9
    ratio: 1.25     # 5:4
  - name: S10
    ratio: 0.8      # 4:5
  - name: S11
    ratio: 2.333    # 7:3
```

### 1.2 `tolerance`
قيمة التسامح المسموح بها عند مقارنة النسبة المطلوبة مع نسب القوالب. إذا كان الفرق أقل من هذه القيمة، يعتبر التطابق دقيقًا. القيمة الافتراضية `0.05` (أي 5%).
```yaml
tolerance: 0.05
```

### كيفية تعديل هذا الملف:
- **إضافة قالب جديد** : أضف سطرًا جديدًا في قائمة `templates` بالصيغة:
  ```yaml
  - name: S12
    ratio: 2.5
  ```
- **تغيير نسبة قالب موجود** : عدّل قيمة `ratio`.
- **تغيير التسامح** : عدّل قيمة `tolerance` (مثلاً `0.1` إذا أردت تسامح 10%).

> 💡 **نصيحة**: اختر أسماء قصيرة للقوالب (مثل S1, S2) حتى يسهل الرجوع إليها في ملف `config_layout.yaml`.

---

## 2. ملف `config_layout.yaml`

هذا هو الملف الأهم، فهو يحدد شكل التصميم بالكامل. كل قالب يمكن أن يكون له إعداداته الخاصة، ولكن يجب أن يحتوي الملف على قالب افتراضي (`default`) يُستخدم في حال عدم وجود إعدادات مطابقة للقالب المختار.

الهيكل العام للملف:
```yaml
default:
  background: { ... }
  mobile_image: { ... }
  logo: { ... }
  decorative_top_right: { ... }
  decorative_middle: { ... }
  icons: [ ... ]
  text_lines: { ... }
  banner: { ... }

S1:
  # إعدادات خاصة بالقالب S1 (قد ترث من default أو تعدل عليه)

S2:
  # إعدادات خاصة بالقالب S2
```

### 2.1 القسم `background` (الخلفية)
يحدد نوع الخلفية ولونها أو صورتها.

الخيارات:
- **`type`** : `color` (لون واحد)، `gradient` (تدرج)، `image` (صورة).
- **`color`** : إذا كان النوع `color`، يمكنك تحديد اللون بصيغة hex مثل `#FFFFFF` أو اسم لون إنجليزي `white`.
- **`image`** : إذا كان النوع `image`، اسم ملف الصورة (يجب أن يكون في مجلد `assets/`).

مثال:
```yaml
background:
  type: color
  color: "#F5F5F5"
```

### 2.2 القسم `mobile_image` (صورة الموبايل)
يحدد مكان وحجم صورة الموبايل.

الحقول:
- **`image`** : اسم ملف الصورة (افتراضي `mobile.png` داخل `assets/`).
- **`width_pct`** : عرض الصورة كنسبة مئوية من عرض التصميم (رقم بين 0 و 100).
- **`anchor`** : نقطة الارتكاز لتحديد موقع الصورة. القيم الممكنة:
  - `right-center` : المنتصف جهة اليمين (الافتراضي)
  - `left-center` : المنتصف جهة اليسار
  - `center` : الوسط تمامًا
  - `top-right`, `top-left`, `bottom-right`, `bottom-left`
- **`x_offset_pct`** : إزاحة أفقية إضافية كنسبة مئوية من عرض التصميم (قيم موجبة لليمين، سالبة لليسار).
- **`y_offset_pct`** : إزاحة رأسية إضافية كنسبة مئوية من ارتفاع التصميم (قيم موجبة للأسفل، سالبة للأعلى).

مثال:
```yaml
mobile_image:
  image: "mobile.png"
  width_pct: 40
  anchor: "right-center"
  x_offset_pct: -5   # إزاحة لليسار قليلاً
  y_offset_pct: 0
```

### 2.3 القسم `logo` (الشعار الرئيسي)
مشابه لصورة الموبايل، لكن عادةً يكون في أعلى اليسار.

الحقول:
- **`image`** : اسم ملف الشعار (افتراضي `logo.png`).
- **`width_pct`** : عرض الشعار كنسبة مئوية.
- **`anchor`** : نقطة الارتكاز (مثل `top-left`).
- **`x_offset_pct`** ، **`y_offset_pct`** : إزاحات إضافية.

مثال:
```yaml
logo:
  image: "logo.png"
  width_pct: 15
  anchor: "top-left"
  x_offset_pct: 5
  y_offset_pct: 5
```

### 2.4 الأقسام الزخرفية `decorative_top_right` و `decorative_middle`
عناصر زخرفية يمكن أن تكون أشكالاً (دائرة، مستطيل) أو صوراً صغيرة.

الحقول المشتركة:
- **`type`** : `shape` أو `image`.
- **`x_pct`** ، **`y_pct`** : موقع العنصر كنسبة مئوية من عرض/ارتفاع التصميم (0 = أقصى اليسار/أعلى، 100 = أقصى اليمين/أسفل).
- **`width_pct`** : حجم العنصر كنسبة مئوية من العرض (للشكل أو الصورة).

إذا كان النوع `shape`:
- **`shape`** : `circle` أو `rectangle`.
- **`color`** : لون الشكل (hex).

إذا كان النوع `image`:
- **`image`** : اسم ملف الصورة داخل `assets/`.

مثال (دائرة حمراء في أعلى اليمين):
```yaml
decorative_top_right:
  type: shape
  shape: circle
  color: "#FF5733"
  width_pct: 8
  x_pct: 92
  y_pct: 8
```

مثال (صورة زخرفية في المنتصف):
```yaml
decorative_middle:
  type: image
  image: "decor_mid.png"
  width_pct: 20
  x_pct: 50
  y_pct: 50
```

### 2.5 قسم الأيقونات `icons`
قائمة من الأيقونات الصغيرة (عادةً مربعة) توضع في أسفل اليسار. كل عنصر في القائمة له نفس حقول الصورة مع `anchor` (عادة `bottom-left`).

الحقول لكل أيقونة:
- **`image`** : اسم ملف الأيقونة داخل `assets/icons/`.
- **`width_pct`** : عرض الأيقونة كنسبة مئوية.
- **`anchor`** : نقطة الارتكاز (عادة `bottom-left`).
- **`x_offset_pct`** ، **`y_offset_pct`** : إزاحات إضافية.

مثال:
```yaml
icons:
  - image: "icon1.png"
    width_pct: 5
    anchor: "bottom-left"
    x_offset_pct: 5
    y_offset_pct: -5   # رفع قليلاً عن الحافة السفلية
  - image: "icon2.png"
    width_pct: 5
    anchor: "bottom-left"
    x_offset_pct: 12
    y_offset_pct: -5
```

### 2.6 قسم النصوص الثابتة `text_lines`
هذا القسم مسؤول عن رسم السطرين الموجودين في منتصف التصميم، مع دعم الخط العريض للأرقام والوحدات.

الحقول:
- **`y_pct`** : موقع السطر الأول عمودياً (نسبة من الارتفاع، مثلاً 0.6 يعني 60% من أعلى الصفحة).
- **`line2_y_pct`** : موقع السطر الثاني عمودياً.
- **`font_size_pct`** : حجم الخط الأساسي كنسبة مئوية من ارتفاع التصميم (مثل 0.03 = 3%).
- **`font_color`** : لون النص (hex).
- **`font_regular`** : اسم ملف الخط العادي (يجب أن يكون داخل `assets/fonts/`).
- **`font_bold`** : اسم ملف الخط العريض.
- **`left_margin_pct`** : الهامش الأيمن (لأن النص عربي يُقرأ من اليمين لليسار) كنسبة من العرض.
- **`gap_between_phrases_pct`** : المسافة بين العبارات (والفاصل "|") كنسبة من العرض.
- **`separator`** : رمز الفاصل بين العبارات (عادة `|`).
- **`line1_phrases`** : قائمة العبارات الثلاث في السطر الأول.
- **`line2_phrases`** : قائمة العبارتين في السطر الثاني.

مثال كامل:
```yaml
text_lines:
  y_pct: 0.6
  line2_y_pct: 0.7
  font_size_pct: 0.03
  font_color: "#000000"
  font_regular: "NotoNaskhArabic-Regular.ttf"
  font_bold: "NotoNaskhArabic-Bold.ttf"
  left_margin_pct: 0.1
  gap_between_phrases_pct: 0.02
  separator: "|"
  line1_phrases:
    - "أول معالج D7100 5G في العالم"
    - "شاشة منحنية ثلاثية الأبعاد 1.5K 120Hz"
    - "بطارية 6500mAh فائقة النحافة"
  line2_phrases:
    - "تحديثات نظام التشغيل لمدة 3 سنوات"
    - "تحديثـــات أمــــان لـمـدة 5 سنوات"
```

> ملاحظة: الأرقام والوحدات (مثل 1.5K, 120Hz, 6500mAh, 3 سنوات) ستظهر تلقائياً بخط عريض.

### 2.7 قسم الشريط العلوي `banner` (اختياري)
يُستخدم عند إضافة اسم المحل. يظهر شريط أسود في أعلى التصميم يحتوي على شعار واسم المحل.

الحقول:
- **`height_pct`** : ارتفاع الشريط كنسبة مئوية من ارتفاع التصميم الكلي.
- **`background_color`** : لون خلفية الشريط (عادة أسود `#000000`).
- **`logo`** : إعدادات الشعار داخل الشريط:
  - **`image`** : اسم ملف الشعار (افتراضي `logo.png`).
  - **`width_pct`** : عرض الشعار كنسبة من **العرض الكلي**.
  - **`x_offset_pct`** ، **`y_offset_pct`** : إزاحات كنسبة من **العرض الكلي** و **ارتفاع الشريط** على التوالي.
- **`shop_name`** : إعدادات اسم المحل:
  - **`font_size_pct`** : حجم الخط كنسبة من **ارتفاع الشريط**.
  - **`color`** : لون النص.
  - **`x_offset_pct`** ، **`y_offset_pct`** : إزاحات.

مثال:
```yaml
banner:
  height_pct: 20
  background_color: "#000000"
  logo:
    image: "logo.png"
    width_pct: 10
    x_offset_pct: 2
    y_offset_pct: 10   # نسبة من ارتفاع الشريط
  shop_name:
    font_size_pct: 0.05
    color: "#FFFFFF"
    x_offset_pct: 15
    y_offset_pct: 10
```

## 3. كيفية تعديل موقع عنصر أو حجمه (ثلاث خطوات عملية)

### مثال 1: تغيير موقع صورة الموبايل
1. افتح ملف `config_layout.yaml`.
2. ابحث عن القالب الذي تستخدمه (مثلاً `default` أو `S2`). ستجد قسم `mobile_image`.
3. عدّل القيم:
   - `x_offset_pct` : لجعل الصورة أكثر يميناً (قيمة موجبة) أو يساراً (سالب).
   - `y_offset_pct` : لرفع الصورة (سالب) أو خفضها (موجب).
   - `width_pct` : لتكبير أو تصغير الصورة.
4. احفظ الملف وأعد تشغيل التوليد.

### مثال 2: تغيير حجم الخط
1. افتح ملف `config_layout.yaml`.
2. ابحث عن قسم `text_lines` في القالب المناسب.
3. غيّر `font_size_pct`. مثلاً من `0.03` إلى `0.04` يجعل الخط أكبر (4% من ارتفاع التصميم).
4. احفظ الملف وأعد التشغيل.

### مثال 3: إضافة قالب جديد S12 في ملفي التكوين
في `config_sizes.yaml` أضف:
```yaml
- name: S12
  ratio: 2.5
```

في `config_layout.yaml` أضف مقطعاً جديداً باسم `S12`، ويمكنك نسخ إعدادات `default` ثم تعديل ما يناسبك:
```yaml
S12:
  mobile_image:
    width_pct: 35
    anchor: "left-center"
    x_offset_pct: 5
  # ... باقي العناصر
```
